  # 设置 HTTP 代理
  git config --global http.proxy http://proxy.server.com:port
  git config --global https.proxy http://proxy.server.com:port

  git config --global --unset http.proxy
  git config --global --unset https.proxy

检查当前代理：

SSH 通常更稳定：

  # 1. 生成 SSH 密钥（如果还没有）
  ssh-keygen -t ed25519 -C "your_email@example.com"
  
  # 2. 查看公钥
  cat ~/.ssh/id_ed25519.pub
  
  # 3. 添加到 GitHub
  # 访问: https://github.com/settings/keys
  # 点击 "New SSH key"，粘贴公钥
  
  # 4. 更改远程 URL
  git remote set-url origin git@github.com:DECARS-AUTO/decarsauto-website.git
  
  # 5. 测试连接
  ssh -T git@github.com
  
  # 6. 推送
  git push origin main

Q: 为什么会连接超时？
A: 可能是防火墙、代理或 GitHub 服务问题。

Q: 如何确认推送成功？
A: 访问 GitHub 仓库页面，查看最新提交时间。

Q: 文档文件必须推送吗？
A: 不是必须的，主要的 CSS 和 JS 已经在之前的提交中。
   文档只是为了项目记录。

Q: GitHub Pages 多久会更新？
A: 通常 1-2 分钟，可在仓库的 Actions 标签查看部署状态。
